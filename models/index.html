# Legend

-Model: This is a Model

-Global: This Model does not require a pipeline to work with, but needs a pipeline at run time.

-Pyot Core: This object is of type Pyot Core and can use any of the Pyot Core API

-Pyot Static: This object is of type Pyot Static and can use any of the Pyot Static API

-param: Available params that can be passed to the Pyot Core object

-pipeline: Provide the pipeline at runtime, receives a string identifying the pipeline (the model e.g. "val")

-query: This Pyot Core object can append a query to the token (e.g. a begin_index to the LoL MatchHistory)

-endpoint: The endpoint that the Pyot Core object will use to make external request to gather the data. Typically a legend for manipulating the default expiration maps of Stores of type Cache.

-bridge: A "bridge" that returns a instance of another Pyot Core object (e.g. when a rune_id is available in the object, there is a high chance that a rune bridge is available to return a Rune object).

-Iterable: Flags the Pyot Object that it is iterable, meaning it can be indexed or iterated in a loop statement (e.g. for mastery in champion_masteries: ... or champion_masteries[0] where champion_masteries is an instance of ChampionMasteries).

-Iterator: The attr of the object that is used as iterator (e.g. in the example above ChampionMasteries in reality is iterating through its masteries attr which is a list).
